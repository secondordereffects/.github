# Security Policy

This policy applies to all repositories under the [github.com/secondordereffects](https://github.com/secondordereffects) organization.

## Scope

This is primarily a content project. Security concerns include:
- Website vulnerabilities (XSS, injection, etc.)
- Malicious content injection in catalog entries
- Dependency vulnerabilities in build tools
- Sensitive data exposure

## Reporting a Vulnerability

**Do NOT:**
- Open a public GitHub issue
- Post about it on social media

**Do:**
- Email us at [hello@lagbase.com](mailto:hello@lagbase.com) with subject "SECURITY: SOE"
- Include detailed steps to reproduce
- Give us reasonable time to respond (48 hours)

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 7 days
- **Fix timeline:** Depends on severity

## Recognition

We appreciate security researchers who help keep our projects safe.
